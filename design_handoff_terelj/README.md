# Handoff: Тэрэлж Ger Resort — гэр амралтын захиалгын систем

> **Зорилго:** энэ HTML прототипийг бодит, олон хэрэглэгч зэрэг ашиглах боломжтой,
> backend-тэй (бодит цагийн захиалга, төлбөр, нэвтрэлт) production вебсайт болгох.

Энэ багц нь **3 баримт бичиг**-ээс бүрдэнэ:

| Файл | Юу агуулсан | Хэнд |
|------|-------------|------|
| `README.md` (энэ) | Дизайн спек — дэлгэц бүр, өнгө, фонт, layout, copy | Frontend хийх хүн / Codex |
| `BACKEND_SPEC.md` | Өгөгдлийн загвар, API, real-time, hold логик, QPay, auth | Backend хийх хүн / Codex |
| `CODEX_INSTRUCTIONS.md` | Codex-д шууд өгөх алхам алхмаар prompt-ууд | Та |

`prototype/` фолдерт ажиллаж буй HTML прототип бүхэлдээ байгаа — нээгээд харж, кодыг нь лавлаж болно.

---

## 1. Дизайн файлуудын тухай

`prototype/` доторх файлууд бол **HTML-ээр хийсэн дизайн лавлагаа** — харагдах байдал, зан төлөвийг харуулсан прототип. Production-д **шууд хуулж тавих код биш**. Даалгавар нь эдгээр дизайныг target codebase-ийн орчинд (React / Next.js / Vue гэх мэт) тухайн орчны жишгээр **дахин бүтээх** явдал. Орчин байхгүй бол доорх "Санал болгох технологи"-г ашигла.

Прототип нь **localStorage + BroadcastChannel** дээр "амьд синк"-ийг дуурайдаг. Production-д үүнийг **бодит backend + WebSocket/SSE**-ээр солих ёстой (`BACKEND_SPEC.md` үз).

## 2. Чанарын түвшин (Fidelity)

**Hi-fi (өндөр нарийвчлал).** Эцсийн өнгө, фонт, зай, интеракцитай. Frontend-ийг pixel-perfect дахин бүтээ. Доорх токенуудыг яг ашигла.

## 3. Санал болгох технологи (орчин байхгүй бол)

- **Next.js 14 (App Router) + TypeScript** — SSR + API routes нэг дор.
- **Tailwind CSS** (доорх токеныг `@theme`/CSS хувьсагч болгон оруул) эсвэл энгийн CSS module.
- **PostgreSQL + Prisma** — захиалга, гэр, хэрэглэгч.
- **Real-time:** Supabase Realtime эсвэл WebSocket (Socket.IO) — reception/web синк.
- **Auth:** ажилтан/админд (NextAuth эсвэл Lucia). Зочинд заавал биш.
- **Төлбөр:** QPay API (Монгол), банкны данс шилжүүлэг гар баталгаажуулалт.
- **Зураг хадгалалт:** S3-нийцтэй (Cloudflare R2 / AWS S3) + CDN.
- Хувилбар: Postgres-гүй MVP бол **Supabase** (DB + Realtime + Storage + Auth нэг дор) хамгийн хурдан.

## 4. Дэлгэцүүд (Screens / Views)

Сайт нь нэг SPA, дотроо 6 route-той. Толгойн баруун дээд талд **Зочин / Reception / Админ** role switch. Hash routing (`#home`, `#book` ...) ашигласан — production-д жинхэнэ route болгож, reception/admin-г **auth-аар хамгаална**.

### 4.1 Landing (`#home`) — нийтэд нээлттэй
- **Зорилго:** танилцуулга, итгэл төрүүлэх, захиалга руу хөтлөх.
- **Layout (split хувилбар):** Hero нь 2 баганат grid (`1.05fr 1fr`, gap 48px) — зүүн талд гарчиг+CTA, баруун талд 4:5 зурагтай карт дээр "X/20 сул" гэсэн хөвөгч карт. Дараа нь: stat зурвас (4 багана), гэрийн төрөл картууд (`auto-fit minmax(230px,1fr)`), хар дэвсгэртэй "live map teaser" (2 багана), үйлчилгээний tiles, "3 алхам" блок, доод CTA.
- **Hero хувилбар (center/banner):** бүтэн өргөн зурган дэвсгэр дээр төвлөрсөн гарчиг (Tweak-аар сэлгэдэг).
- **Гол copy (MN / EN):**
  - Eyebrow: `ТЭРЭЛЖ · ХАН ХЭНТИЙ` / `TERELJ · KHAN KHENTII`
  - H1: `Тэнгэрийн дор амар` / `Rest under the eternal sky`
  - Sub: `Горхи-Тэрэлжийн зүрхэнд, голын эрэг дээр байрлах 20 гэр, модон байшин. Бодит цагийн захиалга — хүссэн гэрээ газрын зураг дээрээс сонго.`
  - CTA: `Гэр сонгох` / `Choose a ger`, хоёрдогч `Үнэ & багц`.
- **Стат:** `20 гэр & байшин`, `1.5 цаг хотоос`, `24/7 ресепшн`, `<5 мин онлайн захиалга`.

### 4.2 Захиалга (`#book`) — гол урсгал
- **Зорилго:** огноо сонгож, амьд газрын зураг дээрээс сул гэр сонгож, 30 мин барих.
- **Layout:** Дээр нь sticky filter bar (`top:74px`) — Check-in, Check-out (native date), Зочид (stepper), баруун талд "сул тоо". Доор нь төрлийн filter chips (`Бүгд / Энгийн / Гэр бүлийн / VIP / Luxe / Модон байшин`).
- **Үндсэн хэсэг:** гэр сонгоогүй үед 1 багана (зураглал бүтэн өргөн). Гэр сонгомогц `1.5fr 1fr` болж, баруун талд **GerDetail** sticky панел гарна.
- **Зураглал — 2 хувилбар (Tweak):**
  - `map`: дээрээс харсан амралтын газрын зураг. Дэвсгэр нь ногоон gradient, дээр нь SVG-ээр гол (Туул), зам, ой. Гэр бүр **GerMarker** (% байрлалтай дугуй marker, дотор нь жижиг гэрийн glyph + ID шошго).
  - `grid`: кино суудал шиг — төрлөөр бүлэглэсэн, мөр болгон нэг ангилал, дөрвөлжин товчнууд (54×48px).
- **GerMarker өнгө = төлөв:** сул=ногоон, барьцаа=шар (pulse), веб захиалсан=улаан, reception=хөх, амарч буй=бөөрөлзгөнө.
- **GerDetail панел:** дээд талд гол зураг (230px) + 3 жижиг зураг/бичлэг slot, төлвийн badge. Доор: гэрийн нэр, төрөл, үнэ, тоноглол (2 баганат grid, икон+текст), `үнэ × хоног = нийт`, **"30 минут барьж үргэлжлүүлэх"** товч. Сул биш бол блоклосон мэдэгдэл + (барьцаа бол) HoldTimer countdown.
- **Hold үйлдэл:** `TJ.hold()` — сул бол захиалга үүсгэж checkout руу шилжинэ; завсар хэн нэгэн авсан бол toast анхааруулга.

### 4.3 Checkout (`#checkout`)
- **Зорилго:** мэдээлэл авах, нэмэлт үйлчилгээ, төлбөр, баталгаажуулалт.
- **Layout:** `1.4fr 1fr` 2 багана. Зүүн талд: hold banner (countdown эсвэл "дууссан"), зочны мэдээлэл (нэр, утас, тоо), нэмэлт үйлчилгээ (6 ширхэг toggle карт), дараа нь төлбөрийн хэсэг. Баруун талд sticky хураангуй карт (гэрийн зураг, огноо, мөр бүрчилсэн үнэ, нийт, үндсэн товч).
- **Stage-ууд:** `form` → `pay` → `done`.
  - `pay`: QPay (псевдо-QR харуулна) эсвэл Дансаар (банк, данс, нэр, дүн, гүйлгээний утга=захиалгын код). "Төлсөн · баталгаажуулах" дарахад 1.5с "Шалгаж байна…" дараа `done`.
  - `done`: ногоон check, том **захиалгын код** (`TJ-XXXXX`), хураангуй, "Ирэх заавар" / "Дуусгах".
- **Чухал:** баталгаажмагц захиалга `web` төлөвт орж, reception шууд харна.

### 4.4 Reception (`#reception`) — ажилтны самбар (auth шаардлагатай)
- **Зорилго:** бодит цагт бүх гэрийн төлөв харах, биечлэн захиалга бүртгэх, check-in/out, **давхцал хамгаалах**.
- **Layout:** Дээр нь өдөр сонгогч + 5 stat карт (Сул / Барьцаа / Захиалсан / Амарч буй / Орлого). Доор `1.5fr 1fr`: зүүн талд **ReceptionBoard** (төрлөөр бүлэглэсэн том товчнууд, төлвийн өнгө+шошго), баруун талд сонгосон гэрийн **ReceptionAction** панел + **LiveFeed** (сүүлийн 8 үйл явдал).
- **ReceptionAction (төлвөөс хамаарч):**
  - `free` → биечлэн захиалгын форм (нэр, утас, хүн, хоног, төлбөрийн хэлбэр) → "Биечлэн захиалах".
  - `hold` → **блоклосон**: "Онлайн барьцаалагдсан — давхар бүү бүртгэ" + countdown. (← гол давхцал хамгаалалт.)
  - `web`/`walkin` → зочны карт + "Check-in хийх" / цуцлах.
  - `stay` → "Check-out хийх".

### 4.5 Админ (`#admin`) — auth шаардлагатай
- 4 таб: **Тойм** (stat + 7 хоногийн дүүргэлтийн bar chart + сувгийн харьцаа), **Захиалга** (шүүлттэй хүснэгт, устгах), **Гэр & үнэ** (төрөл тус бүрийн үнэ/багтаамж засах — шууд тусна), **Контент** (image-slot-ууд — зураг/бичлэг оруулах).

### 4.6 Заавар (`#guide`)
- 5 алхамт заавар, өнгөний тайлбар, 30-минут hold болон check-in/out мэдээлэл, ажилтанд зориулсан тэмдэглэл.

- Гарчиг/display: **Cormorant Garamond** (serif, 600–700, кирилл дэмжинэ).
- Үндсэн/UI: **Manrope** (400–800).
- Google Fonts-оос: `Cormorant+Garamond` + `Manrope` (subset: cyrillic, cyrillic-ext, latin).
- Eyebrow: 12px, 700, `letter-spacing 0.22em`, UPPERCASE, өнгө `--rust`.

### Зай / радиус / сүүдэр
```
radius: --r-sm 8px, --r 14px, --r-lg 22px, --r-pill 999px
```

### Анимэйшн
- `pulse` — hold (барьцаа) marker/dot цохилно (1.6s).
- `riseIn` — карт/панел гарч ирэх (.6s).
- hover transition 120–160ms; primary товч `translateY(-1px)`.
- `spin` — loading.

### Хээ чимэглэл
- `KheeDivider` — улзий хээ маягийн SVG зураас (хэсэг хооронд).
- `body::before` — нарийн felt-grain texture (SVG noise, multiply, opacity 0.5).

## 6. Икон ба asset

- **Иконууд:** бүгд `components.jsx`-д гар хийцийн 24×24, 2px stroke, `currentColor` SVG (`Icons` объект). Production-д энэ хэв маягийг хадгал эсвэл **Lucide** (ойролцоо) ашиглаж болно.
- **Лого:** `components.jsx`-ийн `Logo` — гэрийн дүрс + улаан хаалга + тооно шугам бүхий SVG. Жинхэнэ лого байвал орлуул.
- **Зураг/бичлэг:** прототипд `<image-slot>` (чирч оруулдаг placeholder). Production-д бодит зураг/бичлэгийг S3/CDN-ээс ачаална. Slot ID-ууд: `hero-photo`, `hero-banner`, `type-<төрөл>`, `ger-<ID>-main`, `ger-<ID>-2/3/vid`.
